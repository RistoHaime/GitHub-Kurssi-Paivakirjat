#  git-oppimispaivakirja: Git projektissa

#### __Mitä hyötyä voisi olla versionhallinnasta, jos kehität projektia yksin?__

Mahdollistaa hyvän dokumentoinnin ja mahdollisuuden siirtyä toiseen haaraan työstämään toimintoa. Toimintojen työstäminen omissa haaroissa estää päähaaran "rikkoutumisen jatkuvasti". Vaikka kehittää projektia yksin, tämän avulla projekti pysyy hallinnassa ja järjestelmällisenä.

#### __Mitä hyötyä voisi olla versionhallinnasta, jos projektissa on useita kehittäjiä?__

Pystytään jakamaan työtehtäviä ja useampi voisi työskennellä eri haaroissa. Versiohallinnalla voidaan myös siirtyä taakseppäin, mikäli tarve. Oman haaran työstämisessä on jälki, joten muut voivat esimerkiksi tarkastaa miten työ etenee ja voidaan korjata toisten virheitä. Tällä ehkäistään myös päähaaran rikkomista. 

#### __Miten järjestäisit projektitiimin versionhallinnan 3-4 hengen ohjelmistoprojektikurssilla? Laadi tiimiläisille lyhyt ohje, miten projektissa toimitaan.__

- Projektissa käytetään päähaarana master
- Kun lähdet työstämään ominaisuutta, tee siitä oma haara.
- Tarkista ennen työskentelyä onko oma haara ajan tasalla.
- Tallenna mieluusti aktiivisesti omaan haaraan.
- Ennen yhdistämistä tarkista ettei oma toiminto riko aikaisemmin toimivaa.
- Kun haara on yhdistetty päähaaraan, poista työskentely haara.
- Kommenttien tulisi vastata mitä on tehnyt ja mitä toiminto tekee.
- Piillota arkaluontoiset asiat, kuten API avaimet, etteivä ne päädy kaikkien nähtäville.
- ÄLÄ POISTA REPOSITORIA!
